II-a

curations

NEW
편성 데이터

큐레이션 1급 엔티티. type으로 region·mood·editorial을 한 구조에. title은 줄바꿈 verbatim. 표지는 cover_spot_id로 해석.

컬럼타입제약설명
idbigintPKidentity. 딥링크는 slug가 담당 → uuid 불필요
typevarchar(16)NOT NULLCHECKregion · mood · editorial
slugvarchar(80)UKNOT NULL안정 식별 · 딥링크 /curations/{slug}
titletextNOT NULL줄바꿈 verbatim, pre-line
subtitletext히어로 sub / 레일 subhead. 상세(06) 응답 생략
leadtext상세 리드문 (region만)
introtext상세 본문 단락 (region만)
cover_spot_idvarchar(32)FK→ spots · ON DELETE SET NULL. null이면 curation_spots[0] 폴백
region_cdvarchar(8)FK→ regions.ldong_regn_cd · type=region 랜덤 풀 스코프
mood_idsmallintFK→ moods.id · type=mood 랜덤 풀 스코프
is_publishedbooleanNOT NULLDEFAULT false미published는 피드 제외 · 404
positionintegerNOT NULLDEFAULT 0홈 슬롯 정렬 (type별)
created_attimestamptzDEFAULT now()생성 시각
updated_attimestamptznow() · onupdate수정 시각

인덱스

  • idx_curations_feed — (type, is_published, position)
  • slug UNIQUE

제약 (수동 검토)

  • ck_curation_type — type IN (region, mood, editorial)
  • FK cover_spot_id → spots SET NULL

타입별 적용

region = title·subtitle·lead·intro·cover·spots / mood = title·subtitle·spots(레일) / editorial = 예약(미사용)

II-b

curation_spots

NEW
손픽 · 순서

손픽 스팟과 순서. 키는 content_id (canonical contentId와 일치). 비면 테마 랜덤으로 채움.

컬럼타입제약설명
curation_idbigintPKFK→ curations.id · ON DELETE CASCADE
content_idvarchar(32)PKFK→ spots.content_id · ON DELETE CASCADE
positionintegerNOT NULL손픽 순서 (그리드/레일)

인덱스

  • idx_curation_spots_order — (curation_id, position)

외래 키

  • curation_id → curations.id CASCADE
  • content_id → spots.content_id CASCADE

키

PK (curation_id, content_id) — 한 큐레이션에 같은 스팟 1회.
