# S1 — 진입 퍼널 (스플래시 · 온보딩 · 로그인 · 권한)

> 2026-06-20 설계 세션. 입력: `docs/mockups/01-splash.html`·`02-onboarding.html`·
> `03-login.html`·`04-permissions.html`, `session-context.md`(잠긴 결정 4),
> `design-brief.md`. 산출물은 **화면/UX/네비 설계**이며,
> DB·API 형식화는 S7·S9, 카카오 OIDC·AppState 재체크 구현은 S10에서 reconcile.

## 0. 설계 원칙 — 게스트-우선 최소 마찰

PicTrip의 핵심 가치(사진 한 장으로 닮은 여행지 찾기)는 **로그인도 위치 권한도
없이** 성립한다. 따라서 진입 퍼널은 **아무것도 강제하지 않는다**: 로그인과 위치
권한은 first-run을 막지 않고, 그것이 정말 필요한 행동(스크랩·마이 탭 / 내주변·
거리순)을 사용자가 시도하는 순간에만 트리거된다.

확정된 first-run 시퀀스:

```
스플래시(01) → [onboarding_seen?] → 미시청: 온보딩(02) → 시청: 홈(05)
온보딩 CTA "사진으로 시작하기" → 사진 선택(08) 직행
온보딩 "건너뛰기"              → 홈(05)
로그인(03)·권한(04)은 first-run 경로에 없음 — 트리거 기반 진입
```

게이트 경계(전 세션 공통으로 참조):

| 게스트 OK (로그인 불필요) | 로그인 필요 (넛지 트리거) |
|---|---|
| 홈 · 큐레이션 상세 · 스팟 상세 | **스크랩(저장) 액션** |
| 사진검색 전과정(선택→분석→결과) | **스크랩 목록 / 전체보기**(마이 안의 계정 데이터) |
| 지도 / 내주변 | (그 외 계정 귀속 데이터 없음) |
| **마이(프로필) 탭 — 게스트 변형으로 열람 가능**(S6 15b) | |

> 마이 탭은 게스트도 **둘러볼 수 있다**(로그인 행 + 빈 보드 + 위치권한/앱버전/약관).
> 풀스크린 로그인(03)은 탭 진입이 아니라 **그 안의 로그인 행/버튼 탭** 시에만 뜬다(A2 결정).

## 1. 화면 01 — 스플래시 (`01-splash.html`)

### 목적
부팅 시 로컬 상태를 hydrate하고 **다음 목적지를 결정하는 분기점**. 사용자에게는
브랜드 로고 화면으로만 보이지만, 기능적으로는 라우팅 게이트다.

### 구성요소
- 검정 배경(`#000`) 풀스크린, 흰색 상태바.
- 중앙: 셀카 캐릭터 라인-SVG(목업의 `splashfig`) + 워드마크 "PicTrip"(800, 38px).
- 인터랙션 요소 없음(버튼·입력 없음).

### 부팅 시퀀스 (병렬 hydrate → 분기)
병렬로 수행하고, 둘 다 끝나면 즉시 분기한다.

1. **`onboarding_seen`** 플래그 읽기 (AsyncStorage, 값 `"1"`).
2. **silent refresh**: `refresh_token`(secure-store) 존재 시 1회 `POST /auth/refresh`
   시도 → 성공이면 새 access를 메모리에 적재(+**슬라이딩 refresh 재민트** — 아래 §3 인증 모델).
   실패(만료·무효·네트워크)면 토큰을 폐기하고 **조용히 게스트로 강등** — 에러 토스트·재시도 없음.

> **슬라이딩 refresh**(S11 §D4, 2026-06-21 승인): refresh 성공 시 서버는 입력 refresh JWT를
> 그대로 에코하지 않고 새 `exp`(now+30d)로 **재민트**(jti·claims 동일). 활동 유저는 무중단,
> 30일 미사용 시에만 재로그인. 즉, 콜드 스타트마다 silent refresh가 성공하면 30일 창이 자동
> 연장된다.

분기:
- `onboarding_seen` 없음 → **온보딩(02)**
- `onboarding_seen` 있음 → **홈(05)** (로그인 성공이든 게스트든 무조건 홈)

> silent refresh 결과는 **분기에 영향을 주지 않는다**(로그인 여부와 무관하게 홈/온보딩).
> 토큰은 이후 화면에서 게이트 트리거가 발동할 때 비로소 의미를 갖는다.

### 표시 시간 / 전이
- 인위적 지연 없음. hydrate 완료 즉시 전이(약 400ms crossfade 허용).
- 강제 업데이트 게이트·서버 헬스체크 **없음**(이번 스코프 비목표).

### 상태
| 상태 | 설명 |
|---|---|
| loading | 화면 자체가 곧 로딩 상태. 별도 스피너 없음 |
| normal | hydrate 완료 → 전이(이 화면은 머무르지 않음) |
| empty | 해당 없음 |
| error | silent refresh 실패는 에러로 노출하지 않음(조용한 게스트 강등). AsyncStorage 읽기 실패 시 `onboarding_seen` 없음으로 간주(=온보딩) |

### 데이터 needs
- 서버 호출: silent refresh(`POST /auth/refresh`) **있을 때만**. 그 외 없음.
- 로컬: AsyncStorage `onboarding_seen`, secure-store `refresh_token`.

### 진입 / 이탈 네비
- 진입: 앱 콜드 스타트(루트). 딥링크 진입 시에도 스플래시를 거쳐 hydrate 후 목적지로.
- 이탈: 온보딩(02) 또는 홈(05). 뒤로가기 불가(루트 교체, replace).

## 2. 화면 02 — 온보딩 (`02-onboarding.html`)

### 목적
사진검색이라는 핵심 가치를 3슬라이드로 보여주고, **즉시 행동(사진 선택)으로 연결**.
최초 1회만 노출.

### 구성요소
- 가로 스와이프 트랙(scroll-snap), 슬라이드 3장. 각 슬라이드 = eyebrow(STEP n) +
  미니 디바이스 프리뷰 + 캡션(h2 + sub).
  카피 SSOT(목업 02 verbatim — eyebrow는 `STEP 1/2/3`만, "· 사진선택/분석중/결과"는 미표시):
  - STEP 1 · h2 "마음에 든 사진을 골라요" / sub "여행 사진이든 인터넷에서 본 풍경이든, 한 장이면 충분해요"
  - STEP 2 · h2 "AI가 분위기를 읽어요" / sub "사진의 색감과 분위기를 분석해 닮은 곳을 찾아요"
  - STEP 3 · h2 "닮은 여행지를 추천받아요" / sub "분위기가 비슷한 곳을 유사도 순으로 보여드려요"
- 우상단 **건너뛰기**(`--ter` 회색).
- 하단 고정 foot: dots 인디케이터(현재 슬라이드 강조) + 메인 CTA **"사진으로 시작하기"**(카메라 아이콘 + 잉크 버튼).
- 미니 디바이스 프리뷰는 정적 목업 이미지(picsum 자리표시 → 실제는 무채색 더미/스팟 이미지). **실데이터 호출 없음.**

### `onboarding_seen` 플래그 규칙
- **명시적 이탈 액션 시점에만** 세팅: CTA "사진으로 시작하기" 탭 **또는** "건너뛰기" 탭.
- **set 타이밍**(S11 §D7): CTA 탭 시 플래그는 08 push **전**에 set한다(네비게이션 직전).
  08에서 곧장 뒤로/취소해도 온보딩이 재노출되지 않도록 **중복 진입 방지**.
- 슬라이드 중간 이탈(앱 강제 종료 등)은 플래그를 세우지 않음 → 다음 부팅에 재노출.
- 저장 위치: AsyncStorage(민감정보 아님, 앱 삭제 시 함께 소멸이 자연스러움). secure-store 아님.

### 상태
| 상태 | 설명 |
|---|---|
| loading | 없음(정적 자산). 폰트/이미지 로드 전 깜빡임은 무채색 스켈레톤으로 흡수 |
| normal | 3슬라이드 스와이프, dots 동기화 |
| empty / error | 해당 없음(서버 비의존) |

### 데이터 needs
- 없음. 전적으로 정적 화면. 프리뷰 이미지는 번들 자산 또는 무채색 더미.

### 진입 / 이탈 네비
- 진입: 스플래시(01)에서 `onboarding_seen` 없을 때만. (설정/재방문으로 다시 볼 경로 없음 — 1회성.)
- 이탈:
  - **CTA "사진으로 시작하기"** → 플래그 세팅 → **사진 선택(08) 직행**. (08에서 뒤로/취소 시 홈으로.)
  - **"건너뛰기"** → 플래그 세팅 → **홈(05)**.
- 뒤로가기: 첫 슬라이드에서 시스템 back은 무시 또는 앱 종료(루트 성격). 슬라이드 간은 스와이프.

### 인터랙션
- 가로 스와이프 / dots 탭으로 슬라이드 이동.
- CTA·건너뛰기 탭 → 위 이탈 규칙.

## 3. 화면 03 — 로그인 (`03-login.html`)

### 목적
3종 소셜(카카오·구글·애플) OIDC 로그인. 게스트-우선이므로 **트리거 기반**으로만 등장.

### 표면(surface) — 단일 컴포넌트, 2모드
같은 로그인 컴포넌트를 컨텍스트에 따라 다른 표면으로 렌더한다.

| 모드 | 트리거 | 형태 |
|---|---|---|
| **풀스크린(기본=목업 03)** | 마이 게스트 변형의 **로그인 행/버튼 탭**(15b) | 목업 03 그대로(브랜드 심볼 + 워드마크 + 3버튼 + 약관 + 뒤로) |
| **시트(파생 표면)** | 스팟 스크랩 시도 · 기타 인라인 넛지 | 동일 컴포넌트를 하단 시트로 제시("저장하려면 로그인이 필요해요" 헤드라인 + 3버튼 + 약관) |

> **목업 SSOT는 풀스크린 03뿐**이다. 시트는 인라인 넛지용 **파생 표면**(목업 미존재 설계
> 추가) — 같은 03 컴포넌트의 다른 presentation일 뿐 별도 화면 아님.

두 모드 공통: 3소셜 버튼 영역, 약관/개인정보 동의 간주 문구.

### 구성요소
- (풀스크린) 좌상단 뒤로 화살표, 중앙 브랜드(잉크 심볼 72px + 워드마크). **태그라인 카피 없음** — 목업 03의 `.copy` 슬롯은 빈 채로 렌더(지어내지 않음).
- 소셜 버튼 3종(브랜드 색 유지 — 무채색 규칙의 예외):
  - **카카오** — `#FEE500` 배경 / `카카오로 계속하기`
  - **Google** — 흰 배경 + 라인 보더 / `Google로 계속하기`
  - **Apple** — `#111113` 배경 / `Apple로 계속하기`
- 하단 약관 문구: "계속 진행하면 **이용약관** 및 **개인정보처리방침**에 동의하는 것으로 간주돼요." (두 링크 → 법적고지 16)
- 양 플랫폼(iOS·Android) 모두 3버튼 노출(애플 가이드라인 충족 + 일관성).

### 인증 흐름(공통)
```
버튼 탭 → provider 네이티브 인증(카카오톡/구글/애플 시트)
  → id_token(OIDC) 획득
  → POST /auth/{provider} { id_token }
  → 백엔드: id_token 검증 → 유저 upsert → 자체 JWT(access+refresh) 발급
  → 앱: access=메모리, refresh=secure-store 저장
  → 보류 액션 재개 + 직전 의도로 복귀
```

#### 공급자별 id_token 검증 규약 (S11 §D4, 승인)
- **공통**: 서명(JWKS)·`iss`·`aud`·`exp`·`nonce`를 **전수 검증**. `alg:none` 거부.
  유저 식별 키 = **`provider`+`sub`**(이메일을 키로 쓰지 않음 — 변경·재사용 위험).
- **Apple**: `iss=appleid.apple.com` · `aud=`네이티브 **bundleID** · `ES256` · `nonce`는
  요청 raw nonce의 `SHA-256`을 **base64url 무패딩**으로 비교(**hex 비교 금지**). 이름은 **최초 1회**만 제공.
- **Google**: `aud`는 **플랫폼별 client_id 집합**을 허용(iOS/Android/web 각 client_id).
- **Kakao**: 카카오 **OIDC 활성** + `scope=openid` · `iss=kauth.kakao.com` ·
  JWKS = `/.well-known/jwks.json`.

#### 클라이언트 토큰 저장 보강 (S11 §D4, 승인)
- SecureStore refresh 저장은 `WHEN_UNLOCKED_THIS_DEVICE_ONLY` 접근성으로(기기 잠금 시 비노출, 백업/이관 차단).
- **iOS 재설치 시 키체인 잔존 정리**: 앱 신규 설치 감지 시 이전 설치의 키체인 refresh를 1회 삭제(stale 토큰 제거).
- refresh는 **single-flight**: 동시 401 다발 시 refresh를 1회만 실행하고 나머지 요청은 결과를 공유.
- 3종 모두 **id_token 검증으로 통일**(기존 카카오 access_token+`/v2/user/me` 방식은 폐기 → S10 reconcile에서 카카오 OIDC 활성화).
- 인증 모델: **sessions/devices 테이블·refresh_token_enc·세션폐기 머신 폐기**. 로그아웃·탈퇴는 Redis **jti 덴리스트**로만 처리.
- refresh는 **슬라이딩 재민트**(S11 §D4): `/auth/refresh` 성공 시 입력 refresh JWT를 그대로
  에코하지 않고 새 `exp`(now+30d)로 재민트한다(**jti·claims 동일** — 회전이 아니므로 도난탐지·
  reuse-detection은 여전히 없음, denylist 모델 유지). 기존 "입력 refresh 토큰 그대로 에코" 규약은 폐기.
- **denylist fail-open**: Redis 도달 불가 시 denylist 검사는 통과시킨다(가용성 우선). access JWT는
  15분간 무검사(만료까지 유효). Redis 도달성 이상은 **운영 알림**으로 노출한다(보안 잔존 위험 가시화).

### 상태
| 상태 | 트리거 | UX |
|---|---|---|
| **loading** | 탭 직후 ~ 토큰 교환 완료 | 누른 버튼만 인라인 스피너, 3버튼 전체 비활성(중복 탭 방지). 전면 오버레이 없음 |
| **취소** | provider 시트에서 사용자 취소/뒤로 | **무음 복귀** — 에러 아님, 토스트 없음, 버튼 원상복귀 |
| **provider 실패** | 네트워크/provider 오류 | 인라인 에러 "잠시 후 다시 시도해 주세요" + 버튼 재활성 |
| **백엔드 실패** | id_token 검증 실패 / 5xx | 동일 인라인 에러. `err.code` 분기, traceId는 콘솔만 |
| **성공** | JWT 발급 완료 | 시트면 닫고 보류 액션 재개 / 풀스크린이면 직전 화면 복귀 |
| empty | 해당 없음 | — |

- **애플 전용**: 첫 로그인만 이름 제공, 재로그인 시 이름 null이 정상. 이름 없어도 가입 진행(닉네임 = provider default/익명).

### 데이터 needs
- `POST /auth/{provider}` (`provider` ∈ kakao·google·apple), 요청 `{ id_token }`, 응답 자체 JWT(access+refresh) + 유저 프로필 최소 필드.
- `POST /auth/refresh`(스플래시·만료 시 공용).
- 보류 액션 컨텍스트(예: 스크랩하려던 `spot_id`)는 클라이언트 상태로 유지 → 성공 후 재실행.

### 진입 / 이탈 네비
- 진입: ① 스팟 스크랩 시도(시트) ② **마이 게스트 변형(15b)의 로그인 행/버튼 탭**(풀스크린) ③ 기타 인라인 넛지(시트). (마이 탭 진입 자체는 로그인을 띄우지 않음 — 게스트 변형 표시.)
- 이탈(성공): **항상 직전 의도/직전 화면**으로. 별도 랜딩으로 튕기지 않음. 보류 액션(스크랩 등) 즉시 재개.
- 이탈(취소/뒤로): 시트 닫힘 또는 풀스크린 pop → 직전 화면. 게스트 유지.
- 약관/개인정보 링크 → 법적고지(16)(모달/푸시 후 복귀).

### 인터랙션
- 버튼 탭 → 인증 흐름. 로딩 중 재탭 차단. 약관 링크 탭 → 법적고지.

## 4. 화면 04 — 위치 권한 (`04-permissions.html`)

### 목적
위치가 정말 필요한 순간(내주변·거리순)에 **OS 권한 전 priming**으로 맥락을 설명하고,
이미 거부된 경우 회복 경로를 제공. first-run에는 등장하지 않는다.

### 트리거
- **지도/내주변 진입 시**(S5 — 04 풀스크린 priming/denied). 리센터 FAB 재진입도 동일.
- **사진검색 흐름은 04를 띄우지 않는다**(S4): 권한 팝업 금지, 이미 허용됐을 때만 좌표 첨부,
  무GPS면 거리순 칩 자체 숨김 → "거리순 칩 시도" 트리거 없음.
- 트리거 시 현재 OS 권한 상태로 분기:

| OS 권한 | 화면 | 동작 |
|---|---|---|
| `granted` | 화면 없음 | 즉시 위치 사용(lat/lng 호출) |
| `undetermined` | **priming(04①)** | "위치 허용하기" → OS 다이얼로그 / "나중에 할게요" → 복귀(OS 안 띄움) |
| `denied` | **denied(04②)** | "설정 열기" → `Linking.openSettings()` / "둘러보기" → 권한 없이 진행 |

> "나중에 할게요"는 denied로 전이시키지 않는다 — 다음에 다시 priming 가능. OS
> 다이얼로그에서 사용자가 거부를 누른 시점부터 `denied` 상태가 된다.

### 구성요소
- **04① priming**: 잉크 아이콘 서클(위치 십자) + 타이틀 "내 주변 여행지를 보여드릴게요"
  + 본문 "가까운 여행지부터 보여드려요. 위치는 추천에만 쓰고 저장하지 않아요." +
  CTA primary "위치 허용하기" / secondary "나중에 할게요".
- **04② denied**: muted 아이콘 서클(위치 사선 차단) + 타이틀 "위치가 꺼져 있어요" +
  본문 "설정에서 위치를 켜면 내 주변 여행지를 추천해 드려요." + CTA primary "설정 열기"
  / secondary "둘러보기".
- 표면 형태: **목업 04 풀스크린**(priming/denied 2변형)이 canonical. ~~바텀시트 표면~~은
  쓰지 않는다(목업 미존재 + A1에서 S5 지도 진입을 04 풀스크린으로 일원화). 04는 "나중에/
  둘러보기"로 즉시 빠져나갈 수 있어 게스트 흐름을 막지 않는다.

### 상태
| 상태 | 설명 |
|---|---|
| loading | OS 권한 조회는 즉시. 별도 로딩 없음 |
| normal | priming 또는 denied 중 하나 |
| empty | 해당 없음 |
| error | OS 권한 조회 실패 시 priming으로 폴백 |

### 데이터 needs
- 서버 호출 없음. `expo-location`의 권한 상태/요청만.
- 권한 허용 후의 lat/lng는 이 화면이 아니라 **소비처(내주변 `/map/nearby`, 거리순
  정렬)** 에서 파라미터로 사용. 위치는 저장하지 않음(목업 카피와 일치).

### consent 기록 시점 (게스트=OS권한만, 로그인 시 동기화)
- **게스트**: 서버 `user_consents` 레코드 없음. OS 권한만으로 동작, lat/lng는 호출
  파라미터로만 전송·미저장.
- **로그인 유저**: `location` consent = **로그인 직후 OS 위치 권한 상태 스냅샷**을
  서버에 upsert(`granted` / `denied`·`undetermined`는 미생성 또는 `denied`).
- **권한 변경 추적**: 앱 **포그라운드 복귀(AppState active) 시** OS 권한을 재확인,
  서버 consent와 다르면 재동기화. (구현 훅 지점만 본 스펙에서 명시; 실제 feature는
  백로그 "AppState 권한 재체크" → S10/후속.)

#### 약관 동의 버전 수명주기 (S11 §D7, 승인)
- **최초 로그인(가입)**: 03 약관/개인정보 동의는 **가입 시점의 현행 `terms_version`을 기록**한다
  (간주 동의 문구가 가리키는 버전 = 그 시점 게시 버전).
- **약관 갱신 시 재동의**: `terms_version`이 올라가면 기록된 동의 버전과 비교해 **재동의를 요구**하는
  정책을 둔다(스키마·트리거 화면 형식화는 S6 법적고지/S7 `user_consents` 버전 컬럼에서).

### 진입 / 이탈 네비
- 진입: 내주변/거리순 트리거.
- 이탈:
  - priming "위치 허용하기" → OS 결과에 따라 granted(소비처로 진행) / denied(다음 진입 시 04②).
  - priming "나중에" → 권한 없이 직전 화면 복귀.
  - denied "설정 열기" → OS 설정. 복귀 시 AppState 훅이 권한 재확인.
  - denied "둘러보기" → 권한 없이 진행(거리순 비활성 등 degraded).

### 인터랙션
- primary/secondary 버튼 탭 → 위 규칙. "설정 열기"는 `Linking.openSettings()`.

## 5. 네비게이션 그래프 (S1 전체)

```
[콜드스타트]
     │
     ▼
  스플래시(01) ──hydrate──► onboarding_seen?
     │                         │ 없음 → 온보딩(02)
     │                         │ 있음 → 홈(05)
     │
온보딩(02)
   ├ "사진으로 시작하기" ─(flag set)─► 사진 선택(08) ─(뒤로)─► 홈(05)
   └ "건너뛰기"          ─(flag set)─► 홈(05)

[게이트 트리거 — 어느 화면에서든]
  스크랩 시도(게스트) ─► 로그인 시트(03) ─성공─► 스크랩 재개·복귀
                                        └취소─► 무음 복귀
  마이 탭(게스트)      ─► 게스트 변형 마이(S6 15b)  ─[로그인 행/버튼 탭]─► 로그인 풀스크린(03) ─성공─► 마이 복귀
  내주변(권한無)        ─► 권한(04: priming|denied) ─► 지도 or degraded(서울중심) 복귀
```

## 5.1 앱 셸 / 전역 네비게이션 (셸 소유 = S1)

> 셸을 정의하는 세션이 없던 공백을 S1이 소유한다(진입 퍼널과 인접). 화면 세션들은 각자의
> 탭 active 표시만 한다.

- **탭바 = 4탭 플랫**: `홈 · 지도 · 사진 · 마이`(목업 05·11·14 공통). 무채색, active=잉크.
  | 탭 | 루트 화면 | 게스트 |
  |---|---|---|
  | 홈 | 05 큐레이션 피드 | OK |
  | 지도 | 11 내 주변 | OK(권한 04 경유) |
  | **사진** | **모달 런처** — 탭 화면이 아님 | OK |
  | 마이 | 14 마이(게스트=15b 변형) | OK(열람) |
- **사진 탭 = 런처(A3 결정)**: "사진" 슬롯 탭 → 탭바 **없는** push/modal 스택 `08→09→10→07`을
  연다(목업 08·09·10엔 탭바 없음). 자체 탭 화면을 갖지 않으며, 플로우 종료/닫기 시 직전 탭으로
  복귀. (런처 패턴 = 인스타 중앙 + 와 동일.)
- **모달/스택 규칙**: 사진 플로우·로그인(03)·권한(04)·지역피커(12)는 현재 탭 위에 얹히는
  모달/스택. 큐레이션상세(06)·스팟상세(07)는 같은 탭 내 push.

## 6. 다른 세션으로 넘기는 항목

- **S7 (DB)**: `user_consents`(`location` 등) 스키마, jti 덴리스트(Redis) 모델,
  유저 upsert/`user_auth_providers`(kakao·google·apple) 매핑. sessions/devices 폐기 반영.
- **S9 (API)**: `POST /auth/{provider}`(id_token) · `POST /auth/refresh` ·
  로그아웃/탈퇴(jti 덴리스트) 계약 형식화.
- **S10 (reconcile)**: ① 카카오 OIDC 활성화(기존 access_token+`/v2/user/me` →
  id_token 검증으로 전환) ② AppState 위치 권한 재체크 feature 구현 ③ 기존 카카오
  인증 over-engineering(sessions/devices/refresh_token_enc) 제거 마이그레이션.

## 7. 확정된 결정 요약

1. first-run = 최소 마찰: 스플래시 → (온보딩 1회) → 홈. 로그인·권한은 트리거 기반.
2. `onboarding_seen`: AsyncStorage, CTA/건너뛰기 명시적 이탈 시 세팅, 중간 이탈은 재노출.
3. 스플래시: flag + silent refresh 병렬 hydrate, 실패 시 조용한 게스트 강등, 인위적 지연 없음.
4. 온보딩 CTA → 사진 선택(08) 직행, 건너뛰기 → 홈.
5. 로그인: 단일 컴포넌트, **풀스크린(목업 03)=기본 / 시트=인라인 넛지 파생 표면**, 3종 OIDC id_token 통일, 로딩/취소/실패 상태 정의, 성공 시 보류 액션 재개. 브랜드 태그라인 없음.
6. 인증 모델: sessions/devices 폐기, Redis jti 덴리스트. refresh는 **슬라이딩 재민트**(S11 §D4 — exp now+30d, jti·claims 동일, "에코" 폐기, 회전/도난탐지 없음), denylist **fail-open**(access 15분 무검사, Redis 도달성 알림). id_token은 공급자별 서명·iss·aud·exp·nonce 전수검증·alg:none 거부·유저키=provider+sub(Apple base64url nonce/최초1회 이름, Google 플랫폼별 aud, Kakao OIDC). 클라: SecureStore `WHEN_UNLOCKED_THIS_DEVICE_ONLY`·iOS 재설치 키체인 정리·single-flight refresh.
7. 권한: **목업 04 풀스크린** priming(undetermined)/denied 2상태, granted 통과, "나중에"≠denied. 지도 진입이 트리거(사진흐름은 04 미트리거), 바텀시트 표면 폐기.
8. consent: 게스트=OS권한만, 로그인 직후 스냅샷 upsert + 포그라운드 재동기화. 약관 동의는 **가입 시점 현행 `terms_version` 기록 + 갱신 시 재동의**(S11 §D7). `onboarding_seen`은 CTA 탭 시 08 push **전**에 set(중복 진입 방지).
9. **앱 셸(§5.1)**: 4탭 플랫(홈·지도·사진·마이), **사진=모달 런처**(탭 화면 아님), **마이 탭은 게스트 열람 가능**(15b) — 풀스크린 로그인은 마이 안 로그인 행/버튼 탭 시에만.
