## 1. 지도(내 주변) 화면 — `11-map.html`

### 1.1 목적
유저의 현위치(또는 선택 지역) 주변의 KTO 관광 스팟을 지도 핀 + 바텀시트 리스트로
탐색한다. 4-탭 셸의 **지도** 탭. 게스트도 권한 없이 사용 가능해야 한다.

### 1.2 구성요소
- **지도 레이어**: KakaoWebMap (WebView + Kakao JS SDK). `features/map/components/KakaoWebMap.tsx`.
  - **스팟 핀**: 잉크(`#171719`) 물방울 핀 + 흰 테두리. 결과 스팟마다 1개.
  - **현위치 마커**: 파란 점(`--user #2D7DF6`) + halo. **GPS 좌표가 있을 때만** 렌더.
- **상단 헤더**(흰 바, 지도 위 오버레이):
  - 라벨 영역: `현위치` 접두사(조건부) + `{지역명}` + chevron(▾). 탭 → 지역 피커 모달.
  - 상태바(시간/시스템 아이콘)는 `SafeAreaView` 안전영역.
- **GPS 폴백 배너 없음**(목업 11에 미존재 → 발명 surface 제거). 권한 미결정/거부 시
  안내·재요청은 **권한 화면 04**(S1, priming/denied)가 전담한다. degraded(서울 중심) 상태에서
  권한 재진입 경로는 리센터 FAB(§1.4).
- **"이 지역에서 검색" 핀**(상단 중앙 floating pill, 조건부): refresh 아이콘 + 텍스트.
  지도가 마지막 조회 중심에서 임계 이상 이동했을 때만 등장.
- **리센터 FAB**(우측, floating 원형): crosshair 아이콘.
- **바텀시트**: 3-스냅. 그립 + 카테고리 칩 + 스팟 리스트(§1.3).
- **하단 탭바**: 홈 · 지도(active) · 사진 · 마이. (셸은 S6/별도 세션 소관, 여기선 active 표시만.)

> **앵커링 메모**: "이 지역에서 검색" pill과 리센터 FAB는 둘 다 바텀시트 **상단 모서리에
> 앵커**되어 시트 스냅에 따라 같이 움직인다(목업: `bottom: calc(60% + 14px)`).

### 1.3 바텀시트 + 리스트
- **스냅**: peek(~12% — 그립 + 칩만 보임, 지도 최대) / half(~58% — 진입 기본, 칩 + 첫 스팟)
  / full(~92% — 리스트 몰입). gesture-handler/reanimated(Expo SDK 56 핀) 사용.
- **카테고리 칩**(가로 스크롤, 단일선택):
  | 칩 | category 파라미터 | KTO 매핑 |
  |---|---|---|
  | 전체 | (생략) | 전체 |
  | 관광지 | `attraction` | contentTypeId 12 |
  | 음식점 | `food` | contentTypeId 39 |
  | 카페 | `cafe` | 39 + cat3 카페 코드 |
  | 레저 | `leisure` | contentTypeId 28 |
  | 쇼핑 | `shopping` | contentTypeId 38 |
  - 기본 `전체`. 칩 변경 → **같은 중심 좌표 유지**, `category`만 바꿔 재조회.
- **스팟 카드**(거리순, 30개 상한):
  - 큰 사진(`firstImageUrl`, KTO URL — 다운로드 금지) / 없으면 인셋 플레이스홀더.
  - 이름(`title`).
  - 메타 행: 핀 아이콘 + `{동} · {세분 카테고리 라벨} · {거리}`. 카테고리 라벨 = **세분 subtype**
    (해변/식물원 = `lcls_systm3_nm`, reconcile §3.1) — **칩 필터의 6 coarse 버킷과 별개**(B5).
    거리 = 공유 util `formatDistance(dist)`(사진검색 S4와 **동일 함수**: <1km `{정수}m` / 1–10km
    `소수1 km` / ≥10km `정수 km`).
  - overview 첫 줄(`overview` **verbatim**, 있을 때만 — null이면 생략).
  - **혼잡도 칩**(S11 §7-A 재도입, 데이터 있을 때만): `congestion` 값에 따라 무채색 텍스트 칩/톤으로
    `한산`(low) / `보통`(medium) / `붐빔`(high) 표기. **null이면 칩 자체를 숨김**(색상 배지 아님 —
    무채색 칩/톤). 메타 행 또는 사진 코너 중 한 곳에 배치(구현 시 체감 조정).
  - 카드 탭 → 스팟 상세(#07, S3).
  - 핀 탭 ↔ 카드 연동: 지도 핀 탭 시 시트가 half로 올라오며 해당 카드로 스크롤+하이라이트.

### 1.4 인터랙션
- **진입**: GPS 권한 상태로 분기(S1 04 트리거 = "지도/내주변 진입").
  - `granted` → 현위치 중심 + `현위치 · {동}` 라벨 + `/map/nearby` 조회.
  - `undetermined` → **04① priming** 표시. "위치 허용하기"→OS 결과대로(granted=현위치 / denied=다음 진입 시 04②); "나중에 할게요"→서울 시청 중심 degraded + 조회.
  - `denied` → **04② denied** 표시. "설정 열기"→OS 설정; "둘러보기"→서울 시청 중심 degraded + 조회.
  - degraded(서울 중심) 시 현위치 마커 없음, 라벨 `서울 중구`(접두사 없음).
- **패닝(지도 드래그)**: 핀/라벨/리스트 **변화 없음**. 중심이 임계(예: 반경의 ~30%) 이상
  벗어나면 "이 지역에서 검색" pill 등장.
- **"이 지역에서 검색" 탭**: 뷰포트 중심 lat/lng로 재조회 → 핀/리스트 갱신, 라벨을 그 중심의
  역지오코딩 결과로(접두사 제거), pill 숨김.
- **리센터 FAB 탭**: GPS 있으면 현위치로 재센터 + 재조회 + 라벨 `현위치 · {동}` 복귀.
  GPS 없음/거부면 **권한 화면 04 재진입**(미결정→04① / 거부→04②).
- **헤더 라벨 탭**: 지역 피커 모달(#12) 오픈.
- **카테고리 칩 탭**: 단일선택 토글, 재조회.

### 1.5 상태
| 상태 | 지도 | 시트 |
|---|---|---|
| **loading** | 이전 핀 유지(있으면) / 신규면 비움 | 스켈레톤 카드 3~4장(사진+2줄) |
| **normal** | 핀 N개 + (조건부)현위치 마커 | 카드 리스트(거리순) |
| **empty**(반경 내 0건) | 핀 없음, 현위치 마커만 | `이 주변엔 아직 추천 스팟이 없어요` + 보조문 `지도를 옮겨 '이 지역에서 검색'을 누르거나, 다른 지역을 선택해 보세요` |
| **GPS 없음/거부** | (04 priming/denied 표시 후 "나중에/둘러보기" 선택 시) 서울 시청 중심, 현위치 마커 **없음** | 정상 리스트(배너 없음) |
| **error**(네트워크/서버) | 이전 상태 유지 | `주변 정보를 불러오지 못했어요` + `다시 시도` 버튼(동일 파라미터 재호출) |

- **빈 vs 에러 구분**: empty = 정상 응답 + 빈 배열(서버 도달 성공). error = 네트워크 실패 또는
  비-2xx `AppError`. 에러만 `다시 시도` 제공, empty는 위치 변경을 유도.

### 1.6 진입/이탈 네비
- **진입**: 탭바 `지도`.
- **이탈**: 스팟 카드/핀 → 스팟 상세(#07). 헤더 라벨 → 지역 피커(#12, 모달, 스택 유지).
  탭바 → 다른 탭.

## 2. 지역 선택 모달 — `12-region-picker.html`

### 2.1 목적
17 시도 → 시군구 2단 드릴다운으로 지도 중심을 다른 행정구역으로 점프시킨다.
지도 화면 위에 dim + 바텀시트로 표시되는 **모달**(별도 라우트 아님, 지도 위 오버레이).

### 2.2 구성요소
- **dim 오버레이**(반투명, 뒤 지도 비침). 탭 → 닫기(취소).
- **시트**(62% 높이, 라운드 상단):
  - 헤더: 좌상단 X(닫기) + 가운데 `지역 선택`.
  - 본문 2단:
    - **좌측(33%)**: 17 시도 세로 리스트(`regions`, 단일선택). 선택 행 = ink + chevron, 배경 흰색.
    - **우측(67%)**: 선택 시도의 시군구 리스트. 맨 위 `{시도} 전체` 헤드 행 + 시군구 행들.
      선택 행 = ink + bold.
  - 하단 **`검색` CTA**(ink 풀블럭 버튼).

### 2.3 인터랙션 & 적용
- **진입 상태**: 현재 헤더 라벨에 해당하는 시도/시군구가 미리 하이라이트(없으면 첫 시도).
- **시도 행 탭**: 우측을 그 시도의 시군구로 교체. 우측 선택은 `{시도} 전체`로 리셋.
- **시군구 행 / `{시도} 전체` 행 탭**: 선택만 변경(**적용 아님**).
- **`검색` CTA 탭**(목업대로 명시 적용):
  1. 모달 닫기.
  2. 지도를 선택 지역 **중심 좌표**로 재센터(`regions-tree`의 centroid).
  3. 그 중심 lat/lng + 현재 카테고리 칩으로 `/map/nearby` 재조회.
  4. 헤더 라벨을 선택 지역명으로 갱신(**`현위치` 접두사 제거**).
  5. "이 지역에서 검색" pill 숨김.
- **X / dim 탭**: 변경 없이 닫기.

> **선택 모델 = 단일선택**. 재센터 모델상 중심 좌표 하나로 귀결되므로 다중선택은 비성립.
> `{시도} 전체` = 시도 centroid(시도 전체 스팟 평균)로 재센터.
>
> **목업 대조(B4)**: 목업 12는 `{시도}전체`를 헤더 스타일(`areahead`)로 그렸지만 **탭 가능한
> 선택 행**으로 구현한다(점+반경 모델에선 "시도 전체"가 유효한 단일 중심) — 헤더 스타일은
> 시각적 구분일 뿐. 좌측 시도는 **17개**(canonical; 목업이 12개만 보인 건 정적 스크롤 트렁케이션).

### 2.4 상태
| 상태 | 처리 |
|---|---|
| **loading**(트리 fetch) | 좌/우 리스트 스켈레톤. 트리는 정적이라 캐시 후 즉시 표시 일반적 |
| **normal** | 2단 리스트 |
| **empty** | (비해당 — 17 시도/시군구는 항상 존재) |
| **error**(트리 fetch 실패) | 시트 본문에 `지역 목록을 불러오지 못했어요` + `다시 시도` |

### 2.5 진입/이탈 네비
- **진입**: 지도(#11) 헤더 라벨 탭.
- **이탈**: `검색`(적용 후 지도로) / X / dim(취소 후 지도로).

## 3. 데이터 needs (비공식 스케치 — S9에서 형식화)

> 화면 관점의 data need만. 엔드포인트 형태는 S9 API 계약에서 확정.

### 3.1 `GET /map/nearby?lat&lng&radius&category`
- 입력: `lat`, `lng`(필수), `radius`(기본 3000, m), `category`(선택, NearbyCategory).
- 출력(거리순): `[{ contentId, title, firstImageUrl, addr1, mapx, mapy, dist(m), category, overview, congestion }]`.
- 정렬 거리 오름차순, 상한 30. bbox 프리필터 → haversine 거리.
- **reconcile**: ① 기존 응답에서 **`crowd` 필드 제거**(무채색 = 혼잡도 배지 없음).
  → **정정(S11 §7-A)**: `crowd`(과거 reconcile에서 제거)를 `congestion`으로 **재도입**한다 —
  `"low"|"medium"|"high"|null`(KTO 15128555 집중률 예측의 향후30일 상대집중률 v를 0~100으로 환산:
  v<34→low / 34~66→medium / >66→high, 없으면 null=칩 숨김). 무채색 텍스트 칩/톤으로 표시.
  **필드 권위 정의=S9**, **`spot_concentration` DB 조인=S7**(트렌딩 화면/엔드포인트는 계속 제거,
  데이터만 카드 enrichment).
  ② 응답 `category`는 **coarse 버킷(NearbyCategory)** — 카드는 **세분 라벨**을 보여줘야 하므로
  `lcls_systm3_nm`(세분 subtype) 필드 추가 필요(`spots.lcls_systm3 → lcls_systm_codes.lcls_systm3_nm` 조인;
  S3 §7과 동일). 칩 필터는 coarse `category` 그대로. ③ `firstImage2Url`은 카드 미사용(생략 가능).

### 3.2 `GET /map/region?lat&lng`
- 출력: `{ sido, sigungu, dong, label }`. Kakao `coord2regioncode`, Redis 캐시.
- 헤더 라벨(현위치/패닝검색 후 라벨) 생성용. **그대로 재사용.**

### 3.3 `GET /map/regions-tree` (신규)
- 출력: `[{ regionCode, regionName, centroid{lat,lng}, sigungus:[{ sigunguCode, sigunguName, centroid{lat,lng} }] }]`.
- 지역 피커 좌/우 리스트 + `검색` 시 재센터 좌표.
- **reconcile**: `regions`(17) + `sigungus`(~250) 조인. **centroid 컬럼이 없음** →
  그 구역에 속한 스팟들의 `mapx/mapy` **평균**으로 파생(시도 centroid = 시도 전체 스팟 평균,
  시군구 centroid = 시군구 스팟 평균). 정적이므로 캐시(Redis/메모리). 빈 시군구는 시도 centroid로 폴백.

### 3.4 권한/위치(클라이언트)
- `expo-location`로 GPS 1회 취득(`/map/nearby`는 `lat`/`lng` 기대). 권한 미결정/거부는
  **권한 화면 04**(S1, priming/denied)로 안내, "나중에/둘러보기" 시 폴백(서울 시청 중심, 배너 없음).
  조회 자체 실패(네트워크)는 §1.5 error.
