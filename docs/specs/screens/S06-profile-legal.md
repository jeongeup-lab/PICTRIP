## 1. 화면 14 — 마이 (탭)

### 목적
계정 정보 표면 + 스크랩 진입점 + 앱 설정(위치권한·버전·로그아웃·약관·탈퇴) 허브.
4탭(홈·지도·사진·마이) 중 `마이` 활성.

### 구성요소 (위→아래)
1. **상태바 / 네비바**: 타이틀 "마이"(중앙).
2. **프로필 헤더**: 아바타(62px 원형) · 이름(19px/700) · 이메일(13.5px/sec).
3. 구분 띠(`sep`, 9px inset).
4. **스크랩 섹션**: 헤더 "스크랩"(18px/800) + 우측 `전체보기 ›`(→13). 가로 스크롤 레일 — 정사각 썸네일(96px) · 이름 · 카테고리(ter). 최근 스크랩순, 최대 ~10개.
5. 구분 띠.
6. **설정 그룹 1**: `위치 권한`(아이콘+라벨+상태값+chevron) / `앱 버전`(아이콘+라벨+값 `1.0.0`, chevron 없음).
7. 구분 띠.
8. **설정 그룹 2**: `로그아웃`(아이콘+라벨, chevron 없음).
9. **푸터**: `약관·정책` | `회원 탈퇴` (구분선으로 분리).
10. **탭바**: 홈·지도·사진·마이.

### 상태
- `loading`: 프로필 헤더 placeholder, 스크랩 레일 스켈레톤(썸네일 3~4개 회색). 설정 그룹은 즉시 표시(로컬 데이터).
- `normal`: 로그인 + 스크랩 ≥1. 위 구성요소 전부.
- `empty`(로그인 + 스크랩 0): 15 좌측 변형 — 스크랩 헤더에 `전체보기` 숨김, 빈 보드(테두리 카드) "아직 스크랩한 곳이 없어요" + `둘러보러 가기`(홈 라인-SVG 아이콘) 버튼(→홈 탭).
- `guest`(비로그인): §3 게스트 변형으로 전환.
- `error`(스크랩 로드 실패): 레일 자리에 인라인 "스크랩을 불러오지 못했어요" + 재시도. 프로필/설정은 정상.

### 데이터 needs (S9 입력)
- `GET /users/me` → `{ id, displayName, email, avatarUrl }` (email·avatarUrl nullable).
- `GET /users/me/saved?limit=10` → 레일용 **canonical 카드** 배열 `{ contentId, title, category, firstImageUrl }`.
- 앱 버전 = Expo `Constants`(로컬, API 아님).
- 위치 권한 상태 = `expo-location`(로컬).

### 네비
- 진입: 탭바 `마이`.
- 이탈: `전체보기`→13 / 레일 카드·미리보기→07 / `위치 권한`→OS 설정 / `약관·정책`→16 / 로그인 행(게스트)→03.

### 인터랙션
- `전체보기`: 13 푸시.
- 레일 카드 탭: 07 스팟 상세.
- `위치 권한` 탭: D4(상태→OS 설정).
- `로그아웃` 탭: D5(확인 Alert→게스트 전환, 탭 유지).
- `회원 탈퇴` 탭: D6(이중 Alert).
- `약관·정책` 탭: 16 푸시.

## 2. 화면 13 — 저장(스크랩 그리드)

### 목적
스크랩한 스팟 전체를 2열 그리드로 열람·관리. **14의 `전체보기`로만 진입 → 항상 로그인 상태**(게스트 도달 불가).

### 구성요소
- 네비바: 뒤로(‹) + 타이틀 "스크랩".
- 스크롤 영역: 2열 그리드(카드 48.5%×150px, row-gap 12). 각 카드 = 이미지(KTO URL) · 하단 스크림 · **하트(우상단, 채워짐)** · 이름 오버레이(좌하단).
- 무한 스크롤(커서 페이지네이션).

### 상태
- `loading`: 스켈레톤 카드 그리드(6~8개 회색 박스).
- `normal`: 스크랩 카드 그리드.
- `empty`: (그리드 내에서 전부 해제했을 때) 인라인 빈 상태 "스크랩이 비었어요" + `둘러보러 가기`(→홈). **자동 뒤로가기 하지 않음**(갑작스러운 화면 이탈 방지).
- `error`: 전체 영역 "불러오지 못했어요" + 재시도.

### 데이터 needs
- `GET /users/me/saved?cursor={c}&limit={n}` → `{ items: [<canonical 카드>], nextCursor }` (카드 = `{ contentId, title, category, firstImageUrl }`). 정렬: 스크랩 최근순(`created_at desc`). *현 백엔드는 limit만 — 커서 페이지네이션은 S9 reconcile.*
- 하트 해제: `DELETE /users/me/saved/{contentId}`.

### 네비
- 진입: 14 `전체보기`.
- 이탈: 뒤로→14 / 카드 탭→07 / 빈 상태 CTA→홈 탭.

### 인터랙션
- 카드 탭: 07 스팟 상세.
- 하트 탭: D2(낙관적 제거 + 실행취소 토스트). 실행취소 = `POST /users/me/saved/{contentId}` 재등록. 서버 오류 시 카드 복원 + 에러 토스트.

## 3. 화면 15 — 마이 상태 변형

목업 15는 마이(14)의 두 변형을 보여줌. (14 본문은 `normal`.)

### 3a. 로그인 · 스크랩 없음 (`empty`)
- 프로필 헤더는 14와 동일(로그인 상태).
- 스크랩 섹션: 헤더 "스크랩"만(`전체보기` 없음) + **빈 보드**(테두리 카드, 흰 배경 유지):
  - 텍스트: **"아직 스크랩한 곳이 없어요"** (ter)
  - 버튼: **"둘러보러 가기"**(홈 아이콘) → 홈 탭.
- 설정 그룹·로그아웃·푸터(`약관·정책` | `회원 탈퇴`)는 14와 동일.

### 3b. 비로그인 (게스트, `guest`)
- 프로필 헤더 대신 **로그인 행**: 좌측 사람 아이콘(원형) · **"로그인하기"**(18px/700) · 서브 **"나만의 여행지를 스크랩해 보세요"**(sec) · 우측 chevron → 03 로그인.
- 스크랩 섹션: 헤더 "스크랩"만(`전체보기` 없음) + 빈 보드:
  - 텍스트: **"로그인하고 마음에 든 곳을 스크랩하세요"**
  - 버튼: **"로그인하기"**(로그인 아이콘) → 03.
- 설정 그룹(위치권한·앱버전)은 표시(게스트도 위치권한·버전 의미 있음).
- 로그아웃 그룹 **없음**.
- 푸터: **`약관·정책`만**(회원 탈퇴 없음 — 탈퇴할 계정이 없음).

| 14/15a 빈 보드 텍스트 | 아직 스크랩한 곳이 없어요 |

| 15b 로그인 행 타이틀 | 로그인하기 |
| 15b 로그인 행 서브 | 나만의 여행지를 스크랩해 보세요 |
| 15b 빈 보드 텍스트 | 로그인하고 마음에 든 곳을 스크랩하세요 |
| 15b 빈 보드 버튼 | 로그인하기 |
| 게스트 하트 시트(D3) | 스크랩하려면 로그인이 필요해요 / [로그인하기] |

## 4. 화면 16 — 약관·정책

### 목적
법적 4문서 진입 리스트 + 본문 열람. 14/15 푸터 `약관·정책`로 진입(**게스트도 접근**).

### 구성요소
- 네비바: 뒤로(‹) + 타이틀 "약관·정책".
- 리스트(정적 4행, 각 라벨 + chevron):
  1. 이용약관 — `terms`
  2. 개인정보처리방침 — `privacy`
  3. 위치기반서비스 이용약관 — `location`
  4. 데이터 출처 — `data-sources`

### 본문 뷰 (인앱 WebView, D1)
- 행 탭 → WebView 화면 푸시(헤더 = 문서명, 뒤로 = 리스트).
- URL = `https://pictrip.org/legal/{slug}`.
- **데이터 출처 페이지**: KTO 출처고지 callout 포함(컴플라이언스) — "본 서비스는 한국관광공사 TourAPI 데이터를 사용합니다" 류, 이미지 저작권(`cpyrhtDivCd Type3`, URL 참조) 고지.

### 상태
- 리스트: 정적, 항상 표시(로딩/에러 없음).
- WebView: `loading`(상단 프로그레스/스피너) / `normal` / `error`(로드 실패 → "불러오지 못했어요" + 재시도 + "브라우저로 열기"(`Linking.openURL`)).

### 데이터 needs
- **백엔드 없음.** 4개 `{ slug, title }`는 앱 상수(`features/profile/constants` 등). 본문 = 호스팅 페이지.
- 호스팅 산출물(S8/배포 관심사): `pictrip.org/legal/{terms,privacy,location,data-sources}` 정적 HTML(무채색).

## 5. 횡단 — 인증 / 계정 (S9 입력)

JSend 엔벨로프 `{ data, error, meta }`, 오류는 `AppError` 코드 분기.

| 동작 | 엔드포인트 | 비고 |
|---|---|---|
| 내 프로필 | `GET /users/me` | `{ id, displayName, email?, avatarUrl? }` |
| 스크랩 목록(레일/그리드) | `GET /users/me/saved?cursor=&limit=` | 최근순, 카드=canonical, (커서는 S9 reconcile) |
| 스크랩 추가 | `POST /users/me/saved/{contentId}` | 실행취소·게스트 로그인 후 사용 |
| 스크랩 해제 | `DELETE /users/me/saved/{contentId}` | 낙관적 |
| 로그아웃 | `POST /auth/logout` | refresh `jti` Redis 덴리스트 |
| 회원 탈퇴 | `DELETE /users/me` | cascade(아래) + 토큰 덴리스트 |

- 위 경로·카드 작명은 **백엔드 실제 직렬화와 일치**(`/users/me/saved/{contentId}`,
  카드 `{contentId,title,firstImageUrl,category}`). ~~`/me/saved-spots/{spotId}`·`{spotId,thumbnailUrl}`~~ 폐기.

- 탈퇴 cascade 삭제 대상: `users` 행 + `user_saved_spots` + `user_consents` + `user_auth_providers`.
- 위치권한 상태·앱버전은 **API 없음**(로컬: expo-location / Expo Constants).
- 모듈 배치: 위 계정 동작은 `users` 모듈, 약관은 모듈 불필요(앱 상수 + 외부 호스팅).

## 7. Reconcile 노트 (S7 / S10 입력)

- **스크랩**: 기존 `user_saved_spots` 테이블 그대로 사용. 신규 테이블 불필요.
- **아바타**: `users`에 `avatar_url`(nullable, URL only) 컬럼이 없으면 추가. **이미지 바이트 저장 금지**(KTO/개인정보) — 공급사 프로필 이미지 URL만 참조. `avatarUrl`이 404/널이면 클라가 **모노그램(이름 이니셜)/'여행자' 폴백**으로 렌더(프로필 14·상태 15 신원 표면 적용; 이미지 바이트 저장 금지 유지) — S11 §6.
- **약관**: 백엔드 테이블 **불필요**(D1). 단 `user_consents`의 동의 **버전** 추적은 S1(온보딩/동의) 관심사로 이월 — S6은 현재 호스팅 페이지 열람만 한다.
- **세션/디바이스**: 미사용(로그아웃·탈퇴는 Redis `jti` 덴리스트만, 잠금 결정과 일치).
- **호스팅**: `pictrip.org/legal/*` 정적 페이지 배포는 S8(인프라) 산출물 — Cloudflare/FastAPI 정적 서빙 중 택1(S8 결정).
